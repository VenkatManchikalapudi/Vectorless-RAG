# PDF Summary App – Architecture Diagram

Below is an ASCII diagram representing the high-level flow of the PDF Summary App:

```
+-------------------+         +-------------------+         +-------------------+
|                   |         |                   |         |                   |
|     Frontend      | <-----> |     FastAPI       | <-----> |    Ollama AI      |
|   (React, Vite)   |  REST   |   Backend API     |  HTTP   |  (Summarization,  |
|                   |         |                   |         |     Q&A)          |
+-------------------+         +-------------------+         +-------------------+
         |                             |                             |
         |                             |                             |
         v                             v                             |
+-------------------+         +-------------------+                 |
|                   |         |                   |                 |
|   User Uploads    |         |  PDF Processing   |                 |
|      PDF          |         |  (PyPDF2)         |                 |
|                   |         |                   |                 |
+-------------------+         +-------------------+                 |
         |                             |                             |
         |                             v                             |
         |                   +-------------------+                  |
         |                   |                   |                  |
         |                   |   Chunking        |                  |
         |                   |   (Text Split)    |                  |
         |                   |                   |                  |
         |                   +-------------------+                  |
         |                             |                             |
         |                             v                             |
         |                   +-------------------+                  |
         |                   |                   |                  |
         |                   |   Keyword-Based   |                  |
         |                   |   Search Index    |                  |
         |                   |   (BM25)          |                  |
         |                   +-------------------+                  |
         |                             |                             |
         |                             v                             |
         |                   +-------------------+                  |
         |                   |                   |                  |
         |                   |   SQLite DB       |                  |
         |                   | (SQLAlchemy)      |                  |
         |                   +-------------------+                  |
         |                             |                             |
         |                             v                             |
         |                   +-------------------+                  |
         |                   |                   |                  |
         |                   |   API Endpoints   |                  |
         |                   | (Summarize/Q&A)   |------------------+
         |                   +-------------------+
         |                             |
         v                             v
+-------------------+         +-------------------+
|                   |         |                   |
|   User Views      |         |   User Asks Qs    |
|   Summary         |         |   About PDF       |
|                   |         |                   |
+-------------------+         +-------------------+
```

---

**Legend:**
- User interacts with the React frontend (upload, view summary, ask questions).
- Frontend communicates with FastAPI backend.
- Backend processes PDF, chunks, and indexes text using BM25.
- Ollama AI is used for summarization and Q&A.
- Results are returned to the user via the frontend.
